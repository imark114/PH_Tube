VAR

  • Scope: Function scope. This means a var variable declared inside a function is accessible anywhere within that function, including nested functions. However, if declared outside a function, it becomes globally accessible.
  • Reassignment: Allowed. You can freely update the value of a var variable within its scope.
  • Hoisting: var declarations are hoisted to the top of their scope. This means they are accessible before their actual declaration, but their value remains undefined until the declaration is encountered.

LET

  • Scope: Block scope. This means a let variable is only accessible within the block (e.g., curly braces) in which it is declared, including nested blocks. It is not accessible outside that block.
  • Reassignment: Allowed. Similar to var, you can update the value of a let variable within its scope.
  • Hoisting: let declarations are not hoisted. They are only accessible after their declaration is encountered.

CONST

  • Scope: Block scope, similar to let.
  • Reassignment: Not allowed. Unlike var and let, you cannot change the value of a const variable after its declaration. This helps ensure data integrity and prevents unexpected behavior.
  • Hoisting: Similar to let, const declarations are not hoisted.

NULL

  • Intentionally empty value: Used to explicitly indicate that a variable has no assigned value, but should exist in the code. For example, an array element might be set to null initially, to be filled later
  • Object properties: Can be assigned to object properties to indicate their absence. Accessing a non-existent property on an object will return undefined, while accessing a null property will return null itself.

UNDEFINED

  • Uninitialized variables: When a variable is declared but not assigned a value, its value is undefined. Accessing such a variable will result in a runtime error.
  • Object properties: Accessing a non-existent property on an object will return undefined, unless explicitly set to null.
